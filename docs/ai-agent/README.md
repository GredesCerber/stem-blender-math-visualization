# AI-agent docs — заморожены

**Статус:** заморожены как исторический контекст. **Не использовать буквально.**

## Актуальный источник истины

Для любой работы над методичкой открывай:

1. **`docs/metodichka/PLAN.md`** — план новой ознакомительной методички
   (3 акта, DOCX, ≤15 страниц). Это основной документ для исполнителя.
2. **`docs/metodichka/REDESIGN_NOTES.md`** — почему старая методичка
   была переосмыслена и что именно пошло не так.
3. **`README.md`** репозитория — общий вход.
4. **`scripts/`** — реальный код (не дублируется в методичке).

Старая подробная методичка перенесена в `docs/metodichka/old/` и трогать
её не нужно.

## Почему этой папкой не пользоваться

Файлы `AI_AGENT_HANDOFF.md`, `AI_AGENT_EXECUTION_PLAN.md`,
`AI_AGENT_ACCEPTANCE_CHECKLIST.md`, `AI_AGENT_PROMPT_TEMPLATE.md` ссылаются
на давно удалённые документы (`docs/01_*.md`, `docs/guides/*`,
`GETTING_STARTED.md`, `полная_методичка.md`) и описывают уже закрытые задачи.
Ссылки и задачи расходятся с текущим состоянием репозитория.

## Что делать при следующей итерации

Если понадобится возродить «инструкции для ИИ-агента»:

1. Сначала прочитать `docs/metodichka/PLAN.md` — он задаёт новое видение.
2. Перечитать `README.md`, `scripts/`, `tests/` — это реальное состояние.
3. Переписать handoff/execution-plan/checklist с нуля под новую
   ознакомительную методичку, а не пытаться чинить старые файлы.
4. Удалить всё, что не соответствует `PLAN.md`.
