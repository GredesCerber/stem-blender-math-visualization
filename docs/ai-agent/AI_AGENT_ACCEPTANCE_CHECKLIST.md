# AI Agent Acceptance Checklist

Отмечай пункты только после фактической проверки.

---

## A. Репозиторий

- [ ] Нет временных файлов (`~$*.docx`, temp-артефакты).
- [ ] `README.md` соответствует реальной структуре репозитория.
- [ ] Все ссылки в README/docs валидны.

## B. Скрипты

- [ ] `scripts/visualize_function.py` корректно генерирует поверхность.
- [ ] `scripts/generate_surface_mesh.py` корректно генерирует поверхность.
- [ ] `scripts/setup_geometry_nodes_surface.py` корректно создаёт GN-граф.
- [ ] Поведение ошибок и валидации параметров согласовано.
- [ ] Вне Blender скрипты завершаются предсказуемо и информативно.

## C. Blender-результаты

- [ ] Есть финальный `FunctionVisualizer.blend`.
- [ ] Сцена содержит камеру, свет, материал, поверхность.
- [ ] Geometry Nodes параметры управляют формой в реальном времени.

## D. Экспериментальная часть

- [ ] Заполнена минимум 1 серия по амплитуде.
- [ ] Заполнена минимум 1 серия по частоте.
- [ ] Заполнено сравнение минимум 5 функций.
- [ ] Таблицы содержат не шаблон, а реальные наблюдения и выводы.

## E. Ассеты

- [ ] В `assets/renders/` есть минимум 10 осмысленных PNG-файлов.
- [ ] Имена файлов отражают функцию и параметры.
- [ ] Скриншоты пригодны для вставки в отчёт и презентацию.

## F. Методичка и DOCX

- [ ] `docs/metodichka/полная_методичка.md` синхронизирована с кодом.
- [ ] Финальный DOCX открывается без ошибок.
- [ ] В DOCX нет сырого markdown (`**`, служебные символы).
- [ ] Шаблон отчёта со скриншотами актуален.

## G. Перед пушем

- [ ] Коммиты логично сгруппированы (код/документация/ассеты).
- [ ] Сообщения коммитов понятные и предметные.
- [ ] `git pull --rebase` прошёл без конфликтов.
- [ ] `git push` прошёл успешно.

## H. Расширение: поиск путей и прикладные задачи

- [ ] Реализован минимум один алгоритм поиска пути (A* или Dijkstra).
- [ ] Путь строится на поверхности `z = f(x, y)` с понятной функцией стоимости.
- [ ] Маршрут визуализируется в Blender поверх 3D-поверхности.
- [ ] Добавлен docs-раздел с объяснением постановки задачи и запуском.
- [ ] Есть минимум 2 демонстрационных примера (без препятствий и с препятствиями).

---

## Результат

Если в списке есть незакрытые пункты, проект не считается полностью завершённым.
