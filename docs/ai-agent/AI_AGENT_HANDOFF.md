## 3️⃣ Ключевые характеристики

### Компонент: Function Library
**Что это:** Единый регистр всех 5 математических функций  
**Где:** `scripts/function_library.py`  
**Что содержит:**
```python
FUNCTION_REGISTRY = {
    "wave": {...},
    "paraboloid": {...},
    "saddle": {...},
    "ripple": {...},
    "gaussian": {...}
}

class SurfaceConfig(frozen=True):
    function: str
    x_range: tuple
    y_range: tuple
    amplitude: float
    frequency: float
    resolution: int
```

### Компонент: Camera Presets (NEW!)
**Что это:** Оптимальные камеры для каждой функции  
**Где:** `scripts/enhanced_camera_utils.py`  
**Примеры:**
```python
CAMERA_PRESETS = {
    "wave": {
        "location": (13, -11, 8),
        "rotation": (math.radians(60), 0, math.radians(40)),
        "light_energy": 4.0
    },
    # ... и эще 4
}
```

### Компонент: Blend Files
**Что это:** 5 готовых к открытию .blend файлов  
**Находятся в:** корне проекта  
**Что в них:** объекты + камера + свет + материал + сцена **уже заготовлена**  
**Как использовать:** Right-click → Open with Blender

## 12) Вектор развития: формулы + 3D для решения задач (поиск путей и оптимизация)

Следующая итерация должна расширить проект от «визуализации формы» к «решению задач на поверхности».

### 12.1 Что добавить концептуально

1. **Поиск пути на 3D-поверхности**
   - модель поверхности: `z = f(x, y)`;
   - узлы/граф по дискретной сетке;
   - метрика стоимости: длина + уклон + штрафы;
   - алгоритмы: A*, Dijkstra.

2. **Поля стоимости и ограничений**
   - запретные зоны (области недопустимого движения);
   - «дорогие» зоны (высокая стоимость прохода);
   - сценарии: безопасный маршрут, энергоэффективный маршрут, быстрый маршрут.

3. **Визуализация решения в Blender**
   - маршрут как Curve/Polyline поверх поверхности;
   - heatmap стоимости;
   - анимация процесса поиска (опционально).

### 12.2 Учебные кейсы

- Маршрут дрона над рельефом `z = f(x, y)` с ограничением по уклону.
- Поиск безопасного пути «из A в B» с обходом опасных зон.
- Сравнение алгоритмов (A* vs Dijkstra) по длине пути и времени расчёта.

### 12.3 Минимальный технический модуль для этого направления

Рекомендуемая структура:

- `scripts/pathfinding/terrain_graph.py` — дискретизация поверхности в граф.
- `scripts/pathfinding/cost_functions.py` — функции стоимости (длина, уклон, препятствия).
- `scripts/pathfinding/search.py` — A*, Dijkstra.
- `scripts/pathfinding/visualize_path_in_blender.py` — вывод маршрута в Blender.

### 12.4 Критерий готовности этого расширения

Есть демонстрация: пользователь задаёт функцию поверхности + точки старта/финиша, а система строит и отображает маршрут в 3D.

### 12.5 TODO: лабиринт на том же 2D-массиве

Добавить отдельный мини-модуль для работы с лабиринтом на той же дискретной сетке (двумерный массив), чтобы генерация препятствий и поиск пути использовали единое представление данных.

- Функция 1: генерация лабиринта в 2D-массиве (`0` = свободно, `1` = стена).
- Функция 2: поиск пути по этому же 2D-массиву (A* или Dijkstra), без повторной дискретизации.
- Интеграция: конвертация найденного пути в координаты сцены Blender для визуализации.
- Критерий готовности: по фиксированному seed лабиринт и маршрут воспроизводимы, путь проходит только по свободным ячейкам.
