## 2. Зачем этот проект

Современная математика — это не только формулы на бумаге.  
Функции двух переменных `z = f(x, y)` описывают:
- рельеф местности и карты высот,
- электромагнитные поля,
- поверхности давления и температуры в физике,
- финансовые «горы» в экономике.

Проект показывает: **математическую формулу можно «потрогать» — повернуть, покрасить и найти по ней путь**.  
Это переводит абстракцию в конкретный визуальный и интерактивный опыт.

### Образовательный результат

После работы с проектом ученик умеет:
1. Объяснить, что такое `z = f(x, y)` и какие виды поверхностей бывают.
2. Написать Python-скрипт, который строит 3D-объект из математической формулы.
3. Понять, как работает Geometry Nodes — «умный» модификатор Blender.
4. Объяснить алгоритмы поиска пути (Dijkstra, A*) и применить их к реальной задаче.
5. Провести вычислительный эксперимент: изменить параметры и зафиксировать результат.

## 6. Улучшения, которые стоит обсудить

### 6.1 Анимация изменения параметров

Создать анимацию (GIF или MP4), где Amplitude плавно меняется от 0.1 до 3.0.  
Это сильный визуальный эффект для презентации.  
**Реализация:** keyframes через `bpy` или batch_render с линейной интерполяцией.

### 6.2 Веб-экспорт через glTF

Blender умеет экспортировать в `.glb` (WebGL).  
Можно показывать поверхность прямо в браузере через Three.js.  
**Польза:** преподаватель может поделиться ссылкой, а не .blend-файлом.

### 6.3 Интерактивная анимация поиска пути

Показать пошаговый процесс работы алгоритма: узлы «загораются» один за другим.  
**Реализация:** анимировать видимость Curve-объектов или использовать particle system.

### 6.4 Новые функции

- `volcano`: `z = A * exp(-r) * sin(k*r)` — «вулкан»
- `egg_box`: `z = sin(k*x) * sin(k*y)` — «лоток для яиц»
- `hyperboloid`: `z = sqrt(x^2/a^2 + y^2/b^2)` — гиперболоид

## 9. Как запустить проект за 5 минут

```bash
# 1. Открыть финальную сцену
blender FunctionVisualizer.blend

# 2. Быстрый рендер любой функции
blender --background --python scripts/visualize_function.py -- \
  --function wave --amplitude 1.5 --frequency 2 --resolution 80 \
  --output assets/renders/my_wave.png

# 3. Поиск пути без препятствий
blender --background --python scripts/pathfinding/visualize_path_in_blender.py -- \
  --function paraboloid --algorithm astar \
  --start-x -4 --start-y -4 --goal-x 4 --goal-y 4 \
  --output assets/renders/path_demo.png

# 4. Поиск пути с препятствием
blender --background --python scripts/pathfinding/visualize_path_in_blender.py -- \
  --function wave --algorithm dijkstra \
  --obstacle-circle 0,0,1.5 \
  --start-x -4 --start-y -4 --goal-x 4 --goal-y 4 \
  --output assets/renders/path_obstacle.png

# 5. Запуск тестов
python -m unittest discover -s tests
```

## 10. Технические заметки

### Совместимость Blender

| API | Blender 3.6 | Blender 4.x | Blender 5.x |
|---|---|---|---|
| `mesh.calc_normals()` | ✅ | ❌ удалён | ❌ удалён |
| `Material.use_nodes` | ✅ | ✅ | ⚠️ deprecated |
| `ng.interface.new_socket` | ❌ | ✅ | ✅ |
| `ng.inputs.new` / `ng.outputs.new` | ✅ | ⚠️ deprecated | ❌ |

Текущий код поддерживает Blender 4.x и 5.x.  
Для 3.6 потребуется вернуть `mesh.calc_normals()` и `ng.inputs.new()`.

### Структура граничных случаев pathfinding

- Если старт или финиш в блокированной зоне → `ValueError`
- Если путь не найден → `result.success == False`
- Если функция возвращает `inf` или `nan` → `ValueError` в `evaluate_point`
