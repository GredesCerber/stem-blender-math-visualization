# Рабочий лист студента: функции двух переменных и поиск пути

> **Формат:** раздаточный материал для урока 45–90 минут.
> **Требуется:** Blender 3.6+ или готовые файлы `FunctionVisualizer_*.blend` в корне проекта.
> **Уровень:** 10–11 класс / 1 курс.

---

## Как пользоваться листом

1. Часть 1 — теория, решаешь по учебнику/методичке `docs/metodichka/полная_методичка.md`.
2. Часть 2 — практика, открываешь `.blend` файл и крутишь ползунки **Amplitude** и **Frequency**.
3. Часть 3 — проекты, выбираешь один по согласованию с преподавателем.
4. Свои ответы записывай прямо в копию этого файла (или в распечатке).

---

## Часть 1. Теория (10 вопросов)

**В1.** Запиши определение функции двух переменных `z = f(x, y)` своими словами. Приведи один пример из физики или геометрии.

**В2.** Чем отличаются функции **wave** `z = A·sin(k·x)·cos(k·y)` и **ripple** `z = A·sin(k·r)`, где `r = √(x² + y²)`? Какая из них обладает **радиальной симметрией**?

**В3.** Что изменится на графике волны, если увеличить амплитуду `A` в два раза? А если увеличить частоту `k` в два раза? Ответ обоснуй, не глядя в Blender.

**В4.** Функция **saddle** задаётся формулой `z = A·(x² − y²)`. Почему её график называют «седлом»? В каких точках её значение равно нулю?

**В5.** Для **параболоида** `z = A·(x² + y²)` найди единственный экстремум. Это минимум или максимум? Как ответ зависит от знака `A`?

**В6.** Для **гауссианы** `z = A·exp(−(x² + y²)/σ²)` объясни роль параметра `σ`. Что произойдёт с «колоколом», если `σ` увеличить?

**В7.** Что такое **градиент** функции `f(x, y)`? Запиши формулу и объясни, в какую сторону он указывает на поверхности.

**В8.** Дискретная сетка `N × N` покрывает область `[−5, 5] × [−5, 5]`. Чему равен шаг по оси `x`, если `N = 50`? А если `N = 100`? Как выбор сетки влияет на гладкость рендера?

**В9.** В алгоритме A\* стоимость ребра между двумя соседними точками поверхности считается как
`cost = w_len·длина_в_3D + w_slope·|Δz / Δгоризонт|`.
Объясни, почему штраф уклона важен для задачи «маршрут дрона над рельефом».

**В10.** Чем **A\*** отличается от **Dijkstra**? Какая эвристика используется в нашем проекте (подсказка: см. `scripts/pathfinding/search.py`)?

## Часть 2. Практика (5 упражнений в Blender)

> Для каждого упражнения укажи: параметры, скриншот (или описание), вывод из 1–2 предложений.

### Упражнение 1. Знакомство с волной

1. Открой `FunctionVisualizer_Wave.blend` двойным кликом.
2. Зайди в **Modifier Properties** (гаечный ключ справа) → Geometry Nodes.
3. Поставь `Amplitude = 2.0`, `Frequency = 2.0`. Сделай скриншот viewport.
4. Поставь `Amplitude = 4.0`, `Frequency = 2.0`. Сравни высоту гребней.

**Вопрос:** как высота гребней связана с амплитудой? Запиши наблюдение числом: *при A=2 высота примерно ___ единиц, при A=4 — ___ единиц.*

### Упражнение 2. Волна vs Рябь

1. Открой `FunctionVisualizer_Wave.blend`, зафиксируй `A = 2`, `k = 3`.
2. Открой `FunctionVisualizer_Ripple.blend`, тоже `A = 2`, `k = 3`.
3. Сравни формы: где видны «клетки», а где — концентрические окружности?

**Вопрос:** почему при одинаковых `A` и `k` картинки выглядят по-разному? Ответ связан с ответом на В2.

### Упражнение 3. Охота за четырьмя периодами

1. Открой `FunctionVisualizer_Wave.blend`.
2. Подбирай `Frequency` так, чтобы на отрезке `x ∈ [−5, 5]` укладывалось **ровно 4 полных периода** волны.
3. Проверь формулу: для `sin(k·x)` период равен `2π/k`.

**Вопрос:** какое значение `k` ты выбрал(а)? Сходится ли оно с теорией?

### Упражнение 4. Экстремум параболоида

1. Открой `FunctionVisualizer_Paraboloid.blend`.
2. На глаз найди точку минимума. Запиши её координаты `(x, y, z)`.
3. Поменяй знак амплитуды на `−1.0`. Что произошло с формой? Стал ли экстремум максимумом?

### Упражнение 5. Поиск пути на поверхности

1. Открой терминал в папке проекта.
2. Запусти:
   ```
   blender FunctionVisualizer_Wave.blend -b -P scripts/pathfinding/visualize_path_in_blender.py -- --function wave --algorithm astar --start-x -4 --start-y -4 --goal-x 4 --goal-y 4 --output assets/renders/student_wave_astar.png
   ```
3. Открой получившийся PNG.
4. Повтори с `--algorithm dijkstra`. Сравни итоговые отчёты в консоли.

**Вопрос:** различаются ли длина пути и число посещённых узлов у A\* и Dijkstra? Объясни почему.

## Часть 3. Проекты (выбери один)

### Проект A. Своя функция

Придумай собственную формулу `z = f(x, y)`, в которой есть хотя бы 2 параметра. Пример идей:
- `z = A · sin(k·x + φ)`,
- `z = A · (x² − k·y²)`,
- `z = A / (1 + x² + y²)`.

Что нужно сдать:
1. Формулу с объяснением, за что отвечают параметры.
2. Карту значений 3×3 (три варианта амплитуды × три варианта второго параметра) — можно в виде таблицы.
3. Гипотезу: как твоя функция связана с реальным процессом (волна, рельеф, тепло, …).

> Реализацию в Blender делать **не обязательно** — достаточно вручную построить эскизы или использовать `matplotlib`.

### Проект B. Семейство функций

Возьми уже реализованную функцию (например `wave`) и исследуй её **семейство**, меняя один параметр. Минимум 5 значений параметра, для каждого — рендер или скриншот.

Что нужно сдать:
1. Таблицу «параметр → наблюдение» (например, «при `k = 1` видно 1 период, при `k = 3` видно 3 периода»).
2. График зависимости количества гребней/впадин от параметра.
3. Вывод: линейная это зависимость или нет?

### Проект C. Лабиринт и поиск пути

Используй новый модуль `scripts/pathfinding/labyrinth.py`. Задача — сравнить A\* и Dijkstra на разных лабиринтах.

Что нужно сдать:
1. Три лабиринта с разными seed (например 1, 42, 100) — можно ASCII, можно PNG.
2. Для каждого — результат `find_path_in_maze(...)` для обоих алгоритмов: число посещённых узлов, длина пути.
3. Короткий вывод: даёт ли A\* выигрыш в «посещено узлов»? Почему на идеальном лабиринте длины путей совпадают?

**Подсказка по запуску:**
```bash
python - <<'PY'
import sys; sys.path.insert(0, "scripts")
from pathfinding.labyrinth import generate_maze, find_path_in_maze, maze_start_goal, print_maze

for seed in (1, 42, 100):
    maze = generate_maze(21, 21, seed=seed)
    start, goal = maze_start_goal(maze)
    for algo in ("astar", "dijkstra"):
        result = find_path_in_maze(maze, start, goal, algorithm=algo)
        print(f"seed={seed} {algo}: path={len(result.path)} visited={result.visited_nodes}")
PY
```
