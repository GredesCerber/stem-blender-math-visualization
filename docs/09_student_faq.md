# FAQ студента: частые вопросы и ошибки

---

## Запуск и установка

**Q: Как запустить скрипт в Blender?**  
A: Два способа:
1. GUI: Blender → вкладка Scripting → кнопка Open → выбрать `.py` → кнопка Run Script (▶)
2. Терминал: `blender --background --python scripts/visualize_function.py -- --function wave`

---

**Q: Скрипт выдаёт `ModuleNotFoundError: No module named 'bpy'`**  
A: Ты запускаешь скрипт в обычном Python, а не в Blender. `bpy` — это API Blender, он существует только внутри него. Используй `blender --python ...`.

---

**Q: Ошибка `AttributeError: 'Mesh' object has no attribute 'calc_normals'`**  
A: Этот метод удалён в Blender 4+. Если ты видишь эту ошибку в старом коде — удали строку `mesh.calc_normals()`. В нашем проекте это уже исправлено.

---

**Q: Как проверить версию Blender?**  
A: В терминале: `blender --version`. Или в Blender: Help → About Blender.

---

## Параметры и функции

**Q: Какие функции доступны?**  
A: `paraboloid`, `saddle`, `wave`, `ripple`, `gaussian`, `custom`. Список всегда актуален в `function_library.py` → `FUNCTION_REGISTRY`.

---

**Q: Что такое `--resolution`?**  
A: Количество шагов сетки по каждой оси. При `--resolution 60` сетка 60×60 = 3600 граней, вершин 61×61 = 3721. Чем больше — тем плавнее, но медленнее рендер. Для урока хватает 60–80.

---

**Q: Что такое `--amplitude` и `--frequency`?**  
A:
- `amplitude (A)` — «высота» формы. `wave A=2` в два раза «выше», чем `wave A=1`.
- `frequency (k)` — «густота». `wave k=3` имеет в 3 раза больше волн на том же участке.

---

**Q: Я написал `--function parabolid` и получил ошибку.**  
A: Опечатка! Правильно: `paraboloid` (с буквой `o` в конце). Ошибка будет содержать список правильных имён.

## Geometry Nodes

**Q: Как интерактивно менять форму поверхности?**  
A: Открой `FunctionVisualizer.blend` → выбери объект `MathSurface_GN` → Properties (правая панель) → вкладка Modifier (значок гаечного ключа) → раздел GeoNodes_Surface → меняй поля `Amplitude` и `Frequency`.

---

**Q: Что такое Custom Properties?**  
A: Произвольные поля, которые можно добавить к любому объекту в Blender. Мы используем их для хранения пресетов: Properties → Object Properties → раздел Custom Properties.

---

## Pathfinding (поиск пути)

**Q: Ошибка `ValueError: Start-узел находится в блокированной зоне`**  
A: Точка старта попала внутрь препятствия. Измени `--start-x` / `--start-y`, чтобы старт был за пределами круга препятствия.

---

**Q: Ошибка `ValueError: Маршрут не найден`**  
A: Путь заблокирован полностью — финиш недостижим. Причины:
- Слишком большое препятствие перегораживает всю область
- `--blocked-z-gt` слишком низкий (заблокировал большую часть поверхности)
- Старт и финиш в разных «отсечённых» частях графа  
Решение: уменьши радиус препятствия, увеличь порог `blocked-z-gt`, смени координаты.

---

**Q: Маршрут идёт не туда, куда я ожидал**  
A: Алгоритм оптимизирует стоимость, а не визуальную прямоту. По умолчанию:
- `--w-len 1.0` — учитывает длину пути
- `--w-slope 1.0` — учитывает уклон  
Чтобы получить более «прямой» маршрут: `--w-slope 0`. Чтобы максимально избегать уклонов: `--w-slope 5`.

---

**Q: Почему A* посетил больше узлов, чем Dijkstra на волновой поверхности?**  
A: На сложных поверхностях эвристика A* (прямое расстояние до цели) может «обмануться» — реальный путь идёт в обход, а эвристика не знает об этом. Это нормально: A* всё равно находит оптимальный путь, просто иногда пересматривает узлы. На простых поверхностях (парабол оид) A* всегда быстрее.

---

**Q: Как посмотреть, сколько узлов проверил алгоритм?**  
A: В выводе терминала есть строка:  
`[OK] Маршрут найден: N узлов, стоимость=X.XXXX, посещено=Y, проверено рёбер=Z.`  
`посещено` — это количество извлечений из очереди.

---

**Q: Как добавить несколько препятствий?**  
A: Повторяй `--obstacle-circle` несколько раз:
```bash
blender --background --python scripts/pathfinding/visualize_path_in_blender.py -- \
    --function wave \
    --obstacle-circle 0,2,1.0 \
    --obstacle-circle -2,-1,0.8 \
    --obstacle-circle 2,-2,1.2 \
    --start-x -4 --start-y -4 --goal-x 4 --goal-y 4
```

---

**Q: Что значит `--w-risk`?**  
A: Вес штрафа за «опасные зоны» (risk). Сейчас функция risk всегда 0 — не используется по умолчанию. Это задел для будущего: можно реализовать свою функцию риска в `terrain_graph.py`.

## Рендер и материалы

**Q: Рендер чёрный / очень тёмный**  
A: Нет источника света или камеры. Проверь: в сцене должны быть объекты `Sun` (или `SunLight`) и `Camera`. Запусти скрипт заново — `ensure_camera_and_light()` создаст их автоматически.

---

**Q: Поверхность отображается без цвета (серая)**  
A: Материал не назначен или в Viewport включён режим `Solid` вместо `Material Preview` или `Rendered`. Переключи: кнопка в правом верхнем углу Viewport (4 иконки сфер → выбери третью или четвёртую).

---

**Q: Как сохранить рендер в PNG?**  
A: Добавь `--output путь/к/файлу.png` в команду. Или внутри Blender: Render → Render Image (F12), затем Image → Save As.

---

**Q: Маршрут (красная трубка) не видна на рендере**  
A: Возможно, Curve не отображается в рендере. Выдели объект маршрута (`SurfacePath`) → Properties → Object Properties → убедись, что иконки камеры и глаза включены.

## Код

**Q: Что такое `@dataclass(frozen=True)`?**  
A: `dataclass` — декоратор, который автоматически создаёт конструктор и другие методы. `frozen=True` делает объект **неизменяемым**: попытка изменить поле вызовет ошибку. Это защита от случайных ошибок.

---

**Q: Зачем `heapq` в Dijkstra / A\*?**  
A: `heapq` — это **очередь с приоритетом** (минимальная куча). Она всегда даёт элемент с наименьшим приоритетом (наименьшей стоимостью). Без неё пришлось бы каждый раз просматривать все узлы — O(n²) вместо O(n log n).

---

**Q: Почему путь строится «задом наперёд» через `came_from`?**  
A: Во время поиска мы записываем только «откуда пришли». После нахождения финиша идём по этой цепочке в обратную сторону: `goal → ... → start`, затем разворачиваем список. Это стандартный паттерн для всех алгоритмов на графах.

---

**Q: Можно ли добавить свою функцию поверхности?**  
A: Да! В `scripts/function_library.py`:
```python
def _my_function(x: float, y: float, config: SurfaceConfig) -> float:
    return config.amplitude * math.sin(config.frequency * (x + y))

FUNCTION_REGISTRY["my_function"] = _my_function
```
После этого можно использовать `--function my_function`.
