**Показать 5 формул:**

| Формула | Что видим |
|---|---|
| `z = A(x²+y²)` | параболоид — «миска» |

### 15–25 мин | Практика: разбор кода

**Открыть в Blender:** Scripting → `scripts/visualize_function.py`

**Показать три ключевых блока:**

**Блок 1 — формула:**
```python
def _wave(x, y, config):
    k = config.frequency
    return config.amplitude * math.sin(k * x) * math.cos(k * y)
```
«Это и есть наша математика. Python вычисляет её для каждой точки сетки».

**Блок 2 — сетка:**
```python
for j in range(resolution + 1):
    y = y_min + j * step_y
    for i in range(resolution + 1):
        x = x_min + i * step_x
        z = formula(x, y, config)
        vertices.append((x, y, z))
```
«При resolution=80 — это 81×81=6561 вычислений. Python делает это за доли секунды».

**Блок 3 — передача в Blender:**
```python
mesh.from_pydata(vertices, [], faces)
```
«Одна строчка — и Blender строит объект из наших точек».

**Запустить скрипт** (Run Script). Наблюдать, как поверхность появляется в сцене.

```

```

**Интерактив:** изменить прямо в коде `amplitude=1.0` на `amplitude=2.5`, запустить снова. Обсудить изменение.

### 35–43 мин | Поиск пути

**Рассказать сценарий:** «Представьте дрон. Ему нужно добраться из точки A в точку B над рельефом. Крутые горки — опасны. Как найти оптимальный маршрут?»

**Показать рендер:** `path_paraboloid_astar.png` — синяя поверхность, красный маршрут, зелёная сфера (старт), жёлтая сфера (финиш).

**Объяснить идею Dijkstra** (2 минуты):
- Представь, что ты бросаешь камень в воду. Волна расходится от старта во все стороны.
- Первая волна, которая достигнет финиша — это и есть кратчайший путь.
- В алгоритме «волна» — это очередь с приоритетом, а «расстояние» — стоимость пути.

**Запустить в терминале:**
```bash
blender --background --python scripts/pathfinding/visualize_path_in_blender.py -- \
    --function paraboloid --algorithm dijkstra \
    --start-x -4 --start-y -4 --goal-x 4 --goal-y 4
```

**Показать вывод:** «Посещено 13063 узлов». Затем сравнить с A*: «А* посещает меньше — он умнее выбирает направление».

**Показать:** `path_wave_obstacle_dijkstra.png` — маршрут обходит препятствие.

**Вопрос:** «Почему маршрут не идёт по прямой?»
